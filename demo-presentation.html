Architecture Overview

Frontend (Vercel)

  • React 18 + TypeScript
  • Vite build tool
  • Tailwind CSS styling
  • React Router v6
  • Axios HTTP client
  • Socket.IO client
  • FullCalendar integration
  • react-pdf viewer

Backend (Railway/Render)

  • Node.js + Express
  • MongoDB + Mongoose ODM
  • JWT authentication
  • Socket.IO signaling
  • Stripe SDK
  • Multer file upload
  • Swagger API docs
  • Rate limiting + Helmet

API Documentation

Swagger UI available at /api/docs

Auth

6 endpoints: register, login, me, forgot/reset password, 2FA

Users

4 endpoints: profile CRUD, list investors, list entrepreneurs

Meetings

4 endpoints: create, list, respond, cancel

Documents

3 endpoints: upload, list, e-sign

Payments

4 endpoints: deposit, withdraw, transfer, history

WebSocket

6 events: join-room, offer, answer, ice-candidate, toggle-media, end-call

GET /api/health           -> {"status":"ok"}
GET /api/docs              -> Swagger UI
POST /api/auth/register    -> {name, email, password, role}
POST /api/auth/login       -> {email, password}
GET /api/meetings          -> [meeting objects]
POST /api/documents/upload -> multipart/form-data
POST /api/payments/deposit -> {amount}

Deployment

Frontend — Vercel

Auto-deploys from GitHub. Framework: Vite. Environment: VITE_API_URL

Backend — Railway

Node.js + npm start. MongoDB Atlas for database. Environment variables in dashboard

# Deploy commands
git init && git add . && git commit -m "Nexus full stack"
railway login && railway up    # backend
vercel --prod                  # frontend

Live Demo Flow

  1. Open the deployed app URL
  2. Register as Entrepreneur (e.g., sarah@techwave.io)
  3. Register as Investor in another tab (e.g., michael@vcinnovate.com)
  4. Schedule a meeting from Entrepreneur → Investor
  5. Switch to Investor tab, accept the meeting
  6. Click "Join call" to test WebRTC video
  7. Upload a document and e-sign it
  8. Make a mock deposit and check transaction history
